# Циклический барьер

*Барьер* – примитив синхронизации для фиксированного числа потоков-участников, который представляет собой виртуальную границу в коде, пересечь которую потоки могут только вместе.

Барьер реализует единственный метод: `void Arrive()`

Когда поток хочет пройти через барьер, он вызывает этот метод и блокируется до тех пор, пока к барьеру не подойдут все остальные потоки, т.е. до тех пор, пока все они тоже не вызовут `Arrive()`.

Поток, последним вызвавший `Arrive()`, разблокирует все ожидающие потоки, после чего барьер считается пройденным.

Барьер называется *циклическим*, если потоки могут проходить через него многократно, волнами.


## Задание

Реализуйте циклический барьер с помощью условных переменных.

На один раунд потоки должны тратить константное число нотификаций.

Можно считать, что потоки корректно работают с барьером, т.е. метод `Arrive` вызывается из фиксированного набора потоков-участников. В противном случае поведение барьера не определено.

Шаблон решения находится в файле `cyclic_barrier.hpp`
